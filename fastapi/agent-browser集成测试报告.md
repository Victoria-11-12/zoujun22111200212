## 遇到的问题及解决方案

### 1. 编码问题
**问题描述**：Windows 控制台 GBK 编码与 Unicode 字符冲突，导致特殊符号显示乱码
**解决方案**：
- 替换所有 Unicode 符号为文本标签（[OK]/[FAIL]/[WARN]）
- 使用 UTF-8 解码处理命令行输出
- 实现安全的打印函数避免编码错误

### 2. 命令执行问题
**问题描述**：Windows PowerShell 中 "&&" 分隔符不兼容
**解决方案**：
- 使用 ";" 作为命令分隔符
- 统一使用 shell=True 参数执行命令

### 3. 元素定位问题（已解决）
**问题描述**：百度百科页面搜索框无法通过常见CSS选择器定位
**尝试的选择器**：
- `input[name="word"]`
- `#query`
- `input[placeholder*="百科"]`
- `input[type="text"]`

**问题分析过程**：
1. 首先尝试使用JavaScript的`document.querySelector()`方法查找元素，返回null
2. 使用`document.querySelectorAll('input')`查找所有input元素，发现页面只有1个input元素
3. 通过`agent-browser eval`执行JavaScript脚本获取元素属性，但返回值始终为null
4. 分析页面HTML结构，发现百度百科使用React等框架动态渲染，DOM结构复杂

**最终解决方案**：
- 使用 `agent-browser snapshot -i` 命令获取页面交互元素快照
- 通过快照发现搜索框的引用标识：`textbox [ref=e84]`
- 使用 `agent-browser fill @e84 "战狼2"` 填充搜索框
- 使用 `agent-browser click @e71` 点击"进入词条"按钮
- 成功实现搜索功能，跳转到正确的词条页面

### 4. 电影信息提取问题（已解决）
**问题描述**：无法从百度百科词条页面提取电影详细信息
**尝试的方法**：
1. 使用JavaScript查找`.basicInfo-table`等常见选择器，返回null
2. 使用`document.querySelector()`查找剧情简介元素，返回null
3. 使用`document.body.innerText`获取页面文本，但无法准确定位信息

**问题分析过程**：
1. 分析页面HTML结构，发现百度百科使用自定义组件和动态渲染
2. 传统的CSS选择器无法匹配页面元素
3. 页面内容通过JavaScript动态加载，直接eval JavaScript返回null
4. 需要从页面快照中提取文本信息

**最终解决方案**：
- 使用 `agent-browser snapshot` 获取完整页面快照（包含所有文本内容）
- 从快照文本中通过关键词匹配提取信息：
  - 电影名称：查找`heading "战狼Ⅱ"`标记
  - 基本信息：查找"2017年吴京执导的动作电影"等特征文本
  - 剧情简介：查找"《战狼Ⅱ》是由吴京执导"开头的段落
  - 主要演员：查找"主要演员"区域下的演员名字
- 成功提取电影名称、基本信息、剧情简介、主要演员等字段

### 5. 工具封装与集成问题（已解决）
**问题描述**：如何将爬虫功能集成到FastAPI项目中
**方案对比**：
1. **方案1：封装为普通函数**
   - 优点：逻辑清晰、易于控制、性能好
   - 缺点：不能被Agent自动调用
   - 实现：在 `sql_query_stream` 中判断查询结果，为空时调用爬虫函数

2. **方案2：封装为LangChain Tool（最终选择）**
   - 优点：Agent可自主决策、更智能、符合LangChain架构
   - 缺点：需要精心设计提示词引导Agent
   - 实现：创建 `@tool` 装饰的函数，添加到工具列表

**最终解决方案**：
- 使用 `@tool` 装饰器封装 `baike_search_tool` 函数
- 工具返回格式化的电影信息字符串（Agent自动接收）
- 添加到 `user_toolkit = [sql_db_query, baike_search_tool]`
- 修改 SQL_PROMPT 提示词，明确告知Agent：
  - 优先使用 `sql_db_query` 查询本地数据库
  - 只有当本地数据库无结果时，才使用 `baike_search_tool`
  - 不要同时调用两个工具

**工具返回结果传递机制**：
```
用户问题 → Agent决策 → 调用工具 → 工具返回字符串 → Agent拿到结果 → 生成答案
```
- 工具必须返回**字符串**（LangChain要求）
- Agent会自动将工具结果整合到最终答案中
