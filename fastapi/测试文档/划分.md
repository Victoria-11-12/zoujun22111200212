# FastAPI 项目测试划分文档

## 项目结构概览

```
app/
├── models.py              # 数据模型层 (Pydantic)
├── history.py             # 会话历史管理
├── logs.py                # 日志记录模块
├── config.py              # 全局配置
├── tools/                 # 工具层
│   ├── sql_tools.py       # SQL查询工具
│   ├── admin_tools.py     # 管理员工具集
│   └── web_tools.py       # 网络爬虫工具
├── chains/                # LangChain链层
│   ├── user_chains.py     # 用户对话链
│   ├── admin_chains.py    # 管理员对话链
│   ├── chart_chains.py    # 图表生成链
│   └── eval_chains.py     # 评估链
├── agents/                # Agent层
│   ├── sql_agent.py       # SQL查询Agent
│   └── admin_agent.py     # 管理员Agent
├── workflows/             # 工作流层
│   └── chart_workflow.py  # 图表生成工作流
├── services/              # 服务层
│   ├── user_service.py    # 用户服务
│   ├── admin_service.py   # 管理员服务
│   ├── chart_service.py   # 图表服务
│   └── analyst_service.py # 评估服务
└── routers/               # 路由层
    ├── user.py            # 用户路由
    ├── admin.py           # 管理员路由
    ├── chart.py           # 图表路由
    └── analyst.py         # 评估路由
```

## 三、端到端测试 (E2E Tests)

### 3.1 用户对话接口

**文件**: `app/routers/user.py`

| 测试对象 | 测试内容 | 测试类型 |
|---------|---------|---------|
| POST /api/ai/stream | 完整对话流程、流式响应格式 | E2E测试 |

**场景覆盖**:
- 正常对话请求 → 直接回复
- 需要SQL查询的请求 → 数据库查询 → 结果包装回复
- 安全威胁请求 → 警告拦截 → 安全日志记录
- 多轮对话 → 历史上下文保持
- 异常处理 → 错误流式返回

### 3.2 管理员对话接口

**文件**: `app/routers/admin.py`

| 测试对象 | 测试内容 | 测试类型 |
|---------|---------|---------|
| POST /api/admin/ai/stream | 完整管理员对话流程 | E2E测试 |

**场景覆盖**:
- 正常管理操作 → Agent工具调用 → 结果返回
- 批量操作 → 批次创建 → 多操作执行
- 回滚操作 → 数据恢复
- 安全威胁请求 → 警告拦截
- 危险SQL拦截 → 安全检测触发

### 3.3 图表生成接口

**文件**: `app/routers/chart.py`

| 测试对象 | 测试内容 | 测试类型 |
|---------|---------|---------|
| POST /api/chart/generate | 完整图表生成流程 | E2E测试 |

**场景覆盖**:
- 非图表请求 → 友好提示
- 图表请求 → SQL查询 → 代码生成 → 沙箱执行 → HTML返回
- 代码生成失败 → 重试机制 → 最大次数限制
- 沙箱执行失败 → 错误处理 → 日志记录
- 安全代码检测 → 危险代码拦截

### 3.4 质量评估接口

## 四、测试分层总结

### 4.1 单元测试覆盖范围

**纯单元测试(无需外部依赖)**:
- `app/models.py` - 所有Pydantic模型验证
- `app/history.py` - 历史管理逻辑
- `app/tools/admin_tools.py` - check_sql_safety, set_current_admin_name
- `app/workflows/chart_workflow.py` - _extract_python_code_block, _static_eval, 路由函数

**需Mock的单元测试**:
- `app/logs.py` - 所有日志函数(Mock数据库连接)
- `app/tools/sql_tools.py` - sql_db_query(Mock数据库)
- `app/tools/admin_tools.py` - backup_data, create_user, safe_execute_sql, rollback_batch(Mock数据库)
- `app/tools/web_tools.py` - run_agent_command, baike_search_tool(Mock subprocess)
- `app/config.py` - 所有配置项(Mock环境变量)

### 4.2 集成测试覆盖范围

**链层集成(需LLM)**:
- `app/chains/*.py` - 所有Chain的提示词效果、输出格式

**Agent集成(需LLM+数据库)**:
- `app/agents/sql_agent.py` - Agent工具选择、ReAct循环
- `app/agents/admin_agent.py` - 管理员Agent决策、工具调用

**工作流集成(需完整环境)**:
- `app/workflows/chart_workflow.py` - LangGraph状态流转、节点编排

**服务层集成(需LLM+数据库+Docker)**:
- `app/services/*.py` - 业务逻辑编排、多组件协调

### 4.3 E2E测试覆盖范围

**API接口完整流程(需完整系统)**:
- `app/routers/*.py` - 从HTTP请求到响应的完整链路
- 包含：请求验证 → 服务调用 → 外部依赖(LLM/数据库/Docker) → 流式响应
