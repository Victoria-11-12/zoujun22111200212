## 测试断言原则

| 断言类型 | 适用场景 | 说明 |
|---------|---------|------|
| 关键字匹配 | 意图分类链 | LLM 对特定输入返回固定格式标签（如 NEED_SQL、WARNING、IN_CHART），断言标签存在于结果中 |
| 非空检查 | 生成类链 | LLM 生成的自然语言文本无法预知具体内容，只断言不为空，确保 LLM 正常工作 |
| 结构化字段检查 | 评估链 | Pydantic 模型有固定字段（score、dimensions、verdict），断言字段存在且值在有效范围内 |
| 状态字段检查 | 工作流节点 | 断言执行后状态字典包含预期的 key（sql_result、code_raw、eval_pass） |
| 路由结果检查 | 条件路由 | 断言路由函数返回正确的目标节点名或 END |
| SSE 格式检查 | 流式服务 | 断言数据块以 "data: " 开头、可解析为 JSON 且含 "content" 字段、最终含 "[DONE]" 标记 |
| 数据库操作检查 | 持久化操作 | 断言写入后可查询到对应记录、连接能正常建立和关闭 |
