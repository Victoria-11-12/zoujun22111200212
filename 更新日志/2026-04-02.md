### 缓存优化

**实现原理：**
- 使用内存字典存储预测结果，包含数据、时间戳、过期时间
- 基于时间戳的过期机制（TTL - Time To Live）
- 请求时先检查缓存有效性，有效则直接返回，无效则重新计算

**技术细节：**
```python
dark_horses_cache = {
    'data': None,              # 缓存的预测结果
    'timestamp': 0,            # 缓存生成时间（秒级时间戳）
    'expire_seconds': 300      # 缓存有效期：5 分钟
}

# 缓存检查逻辑
current_time = time.time()
if (cache['data'] is not None and 
    current_time - cache['timestamp'] < cache['expire_seconds']):
    return cache['data']  # 缓存命中
else:
    # 重新计算并更新缓存
```

**性能提升：**
- 首次请求：完整计算（约 2-5 秒）
- 后续请求：直接返回缓存（< 100 毫秒）
- 性能提升约 20-50 倍

**适用场景：**
- 数据更新频率低（电影数据相对稳定）
- 计算成本高（7598 条数据的特征工程 + 模型预测）
- 多用户并发访问
