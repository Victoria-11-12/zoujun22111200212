# 2026-04-28 Agent-Browser 集成完成

## 一、集成背景

为增强系统电影信息查询能力，实现本地数据库与外部数据源的互补，本次完成 agent-browser 工具与 FastAPI 项目的深度集成。通过浏览器自动化技术，实现百度百科电影信息的智能搜索与提取。

## 二、核心功能实现

### 2.1 环境搭建与工具安装

- 成功安装 agent-browser 工具（版本 0.26.0）
- 创建临时目录用于浏览器配置文件存储，解决 socket 目录权限冲突
- 验证工具基本功能正常，可正常打开百度百科页面

### 2.2 元素定位方案

针对百度百科页面使用 React 动态渲染、传统 CSS 选择器失效的问题：

- 采用 `agent-browser snapshot -i` 命令获取页面交互元素快照
- 通过 `[ref=eXX]` 标识符定位元素（搜索框 @e84、搜索按钮 @e71）
- 使用 `fill` 和 `click` 命令完成搜索操作

### 2.3 信息提取机制

- 使用 `agent-browser snapshot` 获取完整页面快照
- 从快照文本中通过关键词匹配提取电影信息：
  - 电影名称：识别 heading 标记
  - 基本信息：匹配特征文本（如"2017年吴京执导的动作电影"）
  - 剧情简介：定位剧情段落开头
  - 主要演员：提取演员区域名字列表

### 2.4 LangChain Tool 封装

- 使用 `@tool` 装饰器封装 `baike_search_tool` 函数
- 工具签名：`def baike_search_tool(movie_name: str) -> str`
- 返回格式化的电影信息字符串，符合 LangChain 规范
- 添加到 `user_toolkit` 工具列表，供 Agent 自主调用

### 2.5 Agent 提示词优化

修改 SQL_PROMPT 提示词，明确工具使用规则：

- 优先使用 `sql_db_query` 查询本地数据库
- 仅当本地无结果时，调用 `baike_search_tool`
- 禁止同时调用两个工具

## 三、问题修复记录

### 3.1 Windows 编码问题

**问题**：控制台 GBK 编码与 Unicode 字符冲突，特殊符号显示乱码
**解决**：替换 Unicode 符号为文本标签（[OK]/[FAIL]/[WARN]），使用 UTF-8 解码命令行输出

### 3.2 元素定位问题

**问题**：百度百科页面搜索框无法通过常见 CSS 选择器定位
**尝试的选择器**：
- `input[name="word"]`
- `#query`
- `input[placeholder*="百科"]`
- `input[type="text"]`

**问题分析**：
- 使用 `document.querySelector()` 查找元素返回 null
- 页面仅包含 1 个 input 元素但无法定位
- 百度百科使用 React 框架动态渲染，DOM 结构复杂

**解决**：使用 `agent-browser snapshot -i` 获取页面交互元素快照，通过 `[ref=e84]` 标识符定位搜索框，`[ref=e71]` 定位搜索按钮

### 3.3 电影信息提取问题

**问题**：无法从百度百科词条页面提取电影详细信息
**尝试的方法**：
- 使用 JavaScript 查找 `.basicInfo-table` 等常见选择器返回 null
- 使用 `document.querySelector()` 查找剧情简介元素返回 null
- 使用 `document.body.innerText` 无法准确定位信息

**问题分析**：
- 百度百科使用自定义组件和动态渲染
- 传统 CSS 选择器无法匹配页面元素
- 页面内容通过 JavaScript 动态加载，直接 eval JavaScript 返回 null

**解决**：使用 `agent-browser snapshot` 获取完整页面快照，从快照文本中通过关键词匹配提取电影名称、基本信息、剧情简介、主要演员等字段

### 3.4 工具封装与集成问题

**问题**：如何将浏览器自动化功能集成到 FastAPI 项目中，使 Agent 能够自主调用

**方案对比**：
- **方案1：封装为普通函数**：逻辑清晰但不能被 Agent 自动调用
- **方案2：封装为 LangChain Tool（最终选择）**：Agent 可自主决策，符合 LangChain 架构

**解决**：
- 使用 `@tool` 装饰器封装 `baike_search_tool` 函数
- 工具返回格式化的电影信息字符串
- 添加到 `user_toolkit` 工具列表
- 修改 SQL_PROMPT 提示词，明确告知 Agent 优先使用本地数据库查询，无结果时再调用浏览器自动化工具

## 四、技术架构

### 4.1 工具调用流程

```
用户问题 → Agent 决策 → 工具执行 → 返回结果 → 生成答案
```

### 4.2 Fallback 机制

```
用户查询电影 → SQL Agent 查本地数据库
    ↓ 无结果
调用 baike_search_tool → 百度百科搜索 → 提取信息 → 返回 Agent
```

### 4.3 搜索执行流程

```
打开百度百科 → 获取交互快照 → 定位搜索框 → 填充关键词
    ↓
点击搜索按钮 → 跳转词条页 → 获取页面快照 → 提取电影信息 → 格式化返回
```
