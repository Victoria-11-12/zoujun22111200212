# 2026-04-30 FastAPI 项目架构重构完成

## 一、重构背景

原 `app3.py` 为单体文件（约2000+行），所有功能集中在一个文件中，导致代码可读性差、维护困难、模块耦合度高。本次重构采用分层架构，将代码按职责拆分为多个模块，提升可维护性和可扩展性。

## 二、架构设计

采用分层架构，目录结构如下：

```
app/
├── config.py          # 配置管理（LLM、数据库连接）
├── models.py          # Pydantic 请求/响应模型
├── history.py         # 对话历史管理
├── logs.py            # 日志记录功能
├── agents/            # Agent 定义
│   ├── sql_agent.py   # SQL查询Agent
│   └── admin_agent.py # 管理员Agent
├── chains/            # LangChain 链定义
│   ├── user_chains.py # 用户相关链
│   ├── admin_chains.py# 管理员相关链
│   ├── chart_chains.py# 图表生成链
│   └── eval_chains.py # 评估链
├── tools/             # 工具函数
│   ├── sql_tools.py   # SQL查询工具
│   ├── web_tools.py   # 网络爬虫工具
│   └── admin_tools.py # 管理员工具集
├── routers/           # API路由
│   ├── user.py        # 用户接口
│   ├── admin.py       # 管理员接口
│   ├── chart.py       # 图表接口
│   └── analyst.py     # 评估接口
├── services/          # 业务逻辑层
│   ├── user_service.py    # 用户服务
│   ├── admin_service.py   # 管理员服务
│   ├── chart_service.py   # 图表服务
│   └── analyst_service.py # 评估服务
└── workflows/         # 工作流定义
    └── chart_workflow.py  # 图表生成工作流
```

## 三、核心变更

### 3.1 配置集中管理

- 将 LLM 初始化、数据库连接集中到 `config.py`
- 支持多数据库连接：管理员库、用户只读库、分析师库
- 新增评估模块专用 LLM 配置

### 3.2 模型层独立

- 所有 Pydantic 模型迁移至 `models.py`
- 统一请求/响应数据结构
- 新增评估结果模型 `ResponseEvalResult`、`CodeEvalResult`

### 3.3 Agent 模块化

- `sql_agent.py`：封装 SQL Agent 和工具调用
- `admin_agent.py`：封装管理员 Agent 和权限控制
- 工具与 Agent 解耦，通过 `user_toolkit` 注入

### 3.4 链式逻辑分离

- `user_chains.py`：意图路由链、直接回复链、SQL包装链、警告链
- `admin_chains.py`：管理员意图链、安全警告链
- `chart_chains.py`：绘图判断链、代码生成链
- `eval_chains.py`：回复评估链、代码评估链

### 3.5 工具函数拆分

- `sql_tools.py`：SQL 查询工具（11行）
- `web_tools.py`：百度百科爬虫工具（182行）
- `admin_tools.py`：用户管理、SQL执行、批次回滚（199行）

### 3.6 路由与服务分层

- 路由层（routers/）：仅负责接口定义和参数校验
- 服务层（services/）：处理业务逻辑、流式响应、日志记录
- 实现关注点分离，路由层无业务逻辑

### 3.7 工作流独立

- 图表生成工作流迁移至 `workflows/chart_workflow.py`
- 使用 LangGraph 实现多节点流程：sqlagent → pythonagent → eval → sandbox
- 支持代码静态检查和 Docker 沙箱执行

## 四、功能保持

重构后所有原有功能保持不变：

- 用户对话流式响应（SSE）
- 意图路由（WARNING/NEED_SQL/DIRECT_REPLY）
- SQL Agent 查询本地数据库
- 百度百科爬虫补充查询
- 管理员增删改查与回滚
- 图表生成（pyecharts + Docker沙箱）
- 质量评估模块（对话评估 + 代码评估）
- 完整日志记录（用户、管理员、图表、安全）
